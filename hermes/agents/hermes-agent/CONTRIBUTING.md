# Contributing to Hermes Agent

We welcome contributions to the Hermes Agent project! Whether it's reporting bugs, suggesting features, or submitting code, your input is valuable.

## How to Contribute

### Reporting Bugs

If you find a bug, please check if it's already reported. If not, open a new issue with a clear description of the problem, steps to reproduce it, and your environment details.

### Suggesting Features

Have an idea for a new feature? Open an issue to discuss it before submitting a pull request.

### Submitting Code

1.  **Fork the repository:** Create your own fork of the Hermes Agent repository.
2.  **Clone your fork:** `git clone https://github.com/your-username/hermes-agent.git`
3.  **Create a new branch:** `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/your-bug-fix`
4.  **Make your changes:** Write your code and ensure it follows project conventions.
5.  **Add tests:** Write unit tests to cover your changes.
6.  **Commit your changes:** `git commit -m "feat: Add new feature"` or `git commit -m "fix: Fix bug"`
7.  **Push to your fork:** `git push origin your-branch-name`
8.  **Open a Pull Request:** Submit a pull request to the main repository.

### Code Style

Please adhere to the existing code style, which generally follows [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code and [Standard JavaScript/TypeScript practices](https://standardjs.com/) for JavaScript/TypeScript.
