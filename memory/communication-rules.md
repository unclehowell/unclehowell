# Communication Rules

## CRITICAL — Never send code to the user on Telegram
- NEVER send code blocks, raw code, technical output, or programming language to the user via Telegram.
- The ONLY exception is URLs that the user can click on (must include the http/https prefix and a valid TLD).
- If the agent needs to share code with the user for any reason, it MUST ask permission first.
- This is non-negotiable. It is a security and usability policy.
- All output to the user must be plain human-readable English text, voice clips, or media attachments (images, files).

## General
- Use plain English in all Telegram messages.
- URLs must be complete with protocol (http:// or https://) and valid TLD.
- When delivering files, use the MEDIA: prefix format or provide a clickable link.
