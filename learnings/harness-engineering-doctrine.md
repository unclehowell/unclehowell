# Harness engineering doctrine (agent-first)

Summary of the user’s instruction (2026-04-08):

- Human attention is scarce; tokens are cheap.
- The engineer’s job shifts from writing code to designing the environment, feedback loops, and context.
- AGENTS.md is a short map (~100 lines), not an encyclopedia.
- docs/ is the system of record; anything not in-repo effectively doesn’t exist to the agent.
- Skills should be short, discrete, composable.
- Taste should be enforced mechanically via lints/tests whose messages teach remediation.
- Self-improvement: session logs are signals; friction becomes new docs/skills/lints.
- Build automation confidence one SDLC stage at a time; agents must own the full loop.
