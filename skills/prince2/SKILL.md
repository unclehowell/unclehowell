# PRINCE2 Project Management

## Description
This skill establishes PRINCE2 as the mandatory methodology for all software engineering projects and delegated tasks within the brain's hive mind.

## MANDATORY USE
This skill MUST be applied to ALL delegated work. No work begins until the BRIEF phase is approved by the user.

## The 5 Phase Framework

### PHASE 1: MANDATE
**Purpose:** Capture exact user requirements before any work begins.

User must provide:
- Exact description of what they want to achieve
- Measurable success criteria
- Any constraints (budget, timeline, technical)

### PHASE 2: BRIEF
**Purpose:** Present proposed solution for user approval BEFORE work begins.

Structure:
- Proposed solution description
- 3 Work Packages (each 33.33% = 3 tasks × 11.11%)
- Risk assessment
- Estimated timeline

**User MUST approve before work starts.**

### PHASE 3: PLAN
**Purpose:** Execute work with real-time football-match style commentary.

Progress tracked at:
- 11.11% (task 1 complete)
- 22.22% (task 2 complete)
- 33.33% (Work Package 1 complete)
- 44.44% (task 4 complete)
- ...continues to 100%

### PHASE 4: HIGHLIGHT REPORTS
**Purpose:** Periodic structured updates for longer tasks.

### PHASE 5: DEBRIEF
**Purpose:** Document lessons learned and final status.

Includes:
- Final status table (Planned vs Actual)
- What went well
- What could be improved
- Recommendations for future projects

## Visual Status Format (MANDATORY)

All status updates MUST use this table format with color indicators:

```
┌─────────────────────────────────────────────────────────────────────────┐
│  PROJECT: [Project Name]                              Status: 🟢 Active │
├─────────────────────────────────────────────────────────────────────────┤
│  MANDATE: [Brief description of what we want to achieve]               │
├─────────────────────────────────────────────────────────────────────────┤
│  Work Package     │ 1 (11%) │ 2 (22%) │ 3 (33%) │ 4 (44%) │ 5 (55%)    │
│  ─────────────────┼─────────┼─────────┼─────────┼─────────┼─────────── │
│  Status           │   🟢    │   🟡    │   ⚪    │   ⚪    │   ⚪       │
│  Task             │ Complete│ In Prog │  Pending│  Pending│  Pending  │
└─────────────────────────────────────────────────────────────────────────┘
```

### Color Schema
- 🟢 **Complete** - Task finished, green
- 🟡 **In Progress** - Currently working, yellow
- 🔴 **Blocked** - Stuck, needs help, red
- ⚪ **Pending** - Not started, gray
- 🟣 **On Hold** - Paused, purple

### Project Status Icons
- 🟢 Active - Work ongoing
- 🟡 On Hold - Waiting on user/dependency
- 🔴 Blocked - Need intervention
- ✅ Complete - Finished
- ⚪ Not Started

## Quality Controls
- Quality gates at 33.33% and 66.66%
- Final sign-off against acceptance criteria
- No work starts without BRIEF approval

## Usage
When receiving a new task:
1. Acknowledge receipt
2. Request Mandate clarification if unclear
3. Present Brief for approval
4. Execute with running commentary
5. Deliver Debrief upon completion
